<analysis>**original_problem_statement:**
Build a calculator that takes a customer's stage dimensions (width, depth, height) and turns it into a basic visual render with a parts list. The parts list is sourced from a component list that the user can upload as an Excel file.

PRODUCT REQUIREMENTS:
- The calculator should have a 3D isometric view of the stage.
- The component list will include name, quantity, price, weight, and dimensions (width, depth).
- The calculator should use available parts to build a stage as close as possible to the requested dimensions.
- There should be a choice between inside and outside builds, which use different calculation equations.
- The currency should be in £ Sterling.
- A toggle for metric (meters) and imperial (feet) measurements.
- For whole number metric inputs, prioritize Aludeck 2x1m components if they can create the exact size. Otherwise, use Litedeck (imperial-sized) components.
- The algorithm should prioritize larger deck panels first, then fill gaps with smaller ones.
- A warning should appear if the exact size isn't achievable, showing the actual dimensions created.
- For indoor builds, add stage legs. The leg height should match the requested stage height (accounting for a 25mm deck offset). 4 legs per deck panel.
- For outdoor builds, use a more complex system of leg savers, base jacks, and wood pads. Aludeck cannot be used for outdoor installations.
- An option to add Stage Valance. If selected, it should cover one of the widest sides. For stages under 760mm, this will force the stage height to adjust to match available valance/leg combinations.
- An option to add Steps. This can be one or two sets. The type of step depends on the stage height.
- An option to add Handrail for the back and sides. Adding steps should replace a section of the handrail.
- A prompt should recommend handrails for stages over 570mm.
- The system should detect inventory shortfalls and highlight the affected components in red in the parts list with a warning message.
- A Save Quote button to download a PDF of the parts list.
- An Add to Cart button that calls a generic API endpoint.

**User's preferred language**: English

**what currently exists?**

The calculator implements complex, user-defined logic for component selection, including prioritizing larger panels, handling metric vs. imperial decking, calculating legs differently for indoor/outdoor setups, and managing inventory shortfalls. It also includes Add to Cart and Save Quote functionality.
